# Quick Task 260604-fj3: MYÜTIK verify-echo branch

## Context

MYÜTIK (the customer's agentic messaging platform) now relays inbound WhatsApp to
`https://gymos-edge-webhooks.fly.dev/webhooks/whatsapp`, configured Meta-style:
it signs every request with header `X-Hub-Signature-256` = `sha256=` + hex
HMAC-SHA256 over the raw body, using the same secret value as `whatsapp_app_secret`.
This makes the existing HMAC check in `services/edge-webhooks/src/routes/whatsapp.ts`
(`verifySignature(raw, sigHeader, appSecret)`) pass for both verify and message events.

MYÜTIK's verify step is a **POST** with body `{"event":"verify","challenge":"<uuid>"}`
and it substring-checks the response body for the challenge. The current POST handler
has no echo path — a verify body has no `entry[]`, so it falls through to
`return c.text("OK", 200)`, which does not contain the challenge → MYÜTIK reports
`challenge_not_echoed`. (Meta itself verifies via GET, so the existing GET handshake
is untouched and unaffected.)

## Task 1: Add authenticated verify-echo branch

**Files:** `services/edge-webhooks/src/routes/whatsapp.ts`

**Action:** In the POST `/whatsapp` handler, AFTER the existing
`verifySignature(...)` check (so the request is authenticated) and AFTER the
`payload = JSON.parse(raw)` block (so the body is parsed), and BEFORE the
`entries` loop, insert a branch:

```ts
// MYÜTIK verify handshake: signed POST with {"event":"verify","challenge"}.
// Echo the challenge in the body so MYÜTIK's substring check passes. Authenticated
// (runs only after verifySignature). Harmless to Meta — Meta verifies via GET.
if ((payload as { event?: string }).event === "verify") {
  return c.json({ challenge: (payload as { challenge?: string }).challenge });
}
```

**Constraints:**
- Must sit after `verifySignature` (authenticated) and after `JSON.parse(raw)`.
- Preserve raw-body-first discipline (no change to the `await c.req.text()` →
  `verifySignature` line order).
- TypeScript only; run prettier on the file.

**Verify:**
- `cd services/edge-webhooks && npx tsc --noEmit` exits 0.
- The verify branch appears between the JSON.parse block and the `entries` loop.

**Done:** edge-webhooks tsc clean; verify branch present and authenticated.
