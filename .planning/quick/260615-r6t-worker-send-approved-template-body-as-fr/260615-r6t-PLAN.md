<objective>
Fix outbound template sends that 400 when the member's 24h session window is OPEN.

MYÜTIK's relay (confirmed in myutik-br1 wa.routes.ts:450-482) computes `requiresTemplate` from its OWN view of the session window: window CLOSED → template branch (needs templateName); window OPEN → free-form branch which requires `text`, IGNORES templateName, and returns `400 "text is required for free-form messages"` when no text is present. Our worker currently always sends approved templates as templates regardless of window, so an in-window template 400s.

Fix: in `sendMessage`, when the payload is an approved template AND the window is OPEN, render the template's BODY text (with `{{N}}` vars substituted) and send it as free-form `text`. When the window is CLOSED, keep the existing template send. The approved-template gate (WA-08) stays in force in both cases.

Purpose: coaches can pick an approved template and have it actually deliver whether or not the member is in-window — no more silent MYÜTIK 400s on in-window template sends.

Output: new worker-local renderer + the in-window text branch in the chokepoint, with Vitest coverage.
</objective>
