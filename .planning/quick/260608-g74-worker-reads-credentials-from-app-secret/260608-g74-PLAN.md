<objective>
Make the worker read WhatsApp/MYÜTIK credentials from the framework `app_secrets`
table — the store the staff-web Settings UI writes to — and prefer it over the
worker's own pgcrypto `secrets` table and env. Eliminates credential drift and
the per-credential `fly secrets set` chore.

Purpose: `app_secrets` becomes the single source of truth for credentials. Today
the worker reads a parallel pgcrypto `secrets` table + env that drifts from what
staff set in the UI.

Output: A worker-local `readAppSecretByKey` reader + resolver wiring so
app_secrets is checked first, with current behavior preserved as fallback when
app_secrets is empty or no key material is configured.
</objective>

  <files>services/worker/src/lib/appSecrets.ts, services/worker/src/lib/appSecrets.test.ts</files>
  <behavior>
    - Happy path: with process.env.BETTER_AUTH_SECRET set to a known value and an
      encrypted_value built with the SAME scheme (sha256(material) key, random
      12-byte iv, `v1:iv:ct:tag`), readAppSecretByKey returns the plaintext.
    - No row: db.execute returns `{ rows: [] }` → returns null.
    - No key material: both SECRETS_ENCRYPTION_KEY and BETTER_AUTH_SECRET unset
      → returns null (does not query — short-circuits).
    - Corrupt payload: encrypted_value is garbage / missing `v1:` prefix / wrong
      key → returns null, never throws.
  </behavior>
  <action>
    Create services/worker/src/lib/appSecrets.ts:

    Imports: `import { sql } from "drizzle-orm";` `import { createHash, createDecipheriv } from "node:crypto";` `import type { getDb } from "./db.js";` `import { getLogger } from "./logger.js";`

    Export: `export async function readAppSecretByKey(key: string, db: ReturnType<typeof getDb>): Promise<string | null>`

    Behavior, mirroring packages/core/src/secrets/storage.ts:
    1. Derive material = `process.env.SECRETS_ENCRYPTION_KEY || process.env.BETTER_AUTH_SECRET`.
       If falsy → return null. Emit a ONE-TIME debug log via getLogger().debug
       (so misconfig is visible) gated by a module-level `let _warned = false`
       flag (mirror storage.ts's `_warnedFallback`). Do NOT use the CWD-derived
       fallback that storage.ts uses — for the worker, no material means "not
       configured, fall back to other sources", so return null.
    2. Build the AES key: `const aesKey = createHash("sha256").update(material).digest();`
    3. Query latest-by-key (SINGLE-TENANT — one Neon DB per studio, resolve by
       key alone, no scope/scope_id filter; document this in a comment). Put a
       `// guard:allow-unscoped — app_secrets is studio-global (one Neon DB per studio), no ownableColumns` comment on the query:
       ```ts
       const result = await db.execute(sql`
         SELECT encrypted_value FROM app_secrets
         WHERE key = ${key}
         ORDER BY updated_at DESC
         LIMIT 1
       `);
       ```
    4. Row-shape handling (copy from secrets.ts): `const rows = (result as any)?.rows ?? (result as any);`
       If `!rows || rows.length === 0` → return null.
    5. Decrypt inside try/catch. On ANY failure (bad `v1:` prefix, split missing
       parts, wrong key, tampered tag) return null — NEVER throw (mirror the
       framework readAppSecret swallow):
       ```ts
       try {
         const enc = rows[0].encrypted_value as string;
         if (!enc.startsWith("v1:")) return null;
         const [, ivHex, ctHex, tagHex] = enc.split(":");
         if (!ivHex || !ctHex || !tagHex) return null;
         const d = createDecipheriv("aes-256-gcm", aesKey, Buffer.from(ivHex, "hex"));
         d.setAuthTag(Buffer.from(tagHex, "hex"));
         const pt = Buffer.concat([d.update(Buffer.from(ctHex, "hex")), d.final()]).toString("utf8");
         return pt;
       } catch {
         return null;
       }
       ```
    Add a file-header comment explaining WHY this is reimplemented (worker does
    NOT depend on @agent-native/core; resolveSecret() is request-scoped and a
    headless cron has no request) and the BETTER_AUTH_SECRET key-material note.

    Create services/worker/src/lib/appSecrets.test.ts (copy the vi/mock-db
    pattern from services/worker/src/domain/syncTemplates.test.ts):
    - Mock the logger: `vi.mock("../lib/logger.js"...)` is NOT needed here since

    - Helper to encrypt with the SAME scheme: `createCipheriv("aes-256-gcm", createHash("sha256").update(material).digest(), iv)` with a random 12-byte iv, return `v1:${iv.toString("hex")}:${ct.toString("hex")}:${tag.toString("hex")}`.
    - afterEach: restore/delete process.env.BETTER_AUTH_SECRET and
      SECRETS_ENCRYPTION_KEY so cases don't bleed.
    - Cases (a) happy path returns plaintext, (b) `{ rows: [] }` → null,
      (c) no key material → null (and db.execute NOT called), (d) corrupt
      encrypted_value → null and no throw.
    - mockDb = `{ execute: vi.fn() }` set per-case via mockResolvedValueOnce.

    Run prettier on both files after writing.
