# Quick 260625-d06: Add Location / Class Type / Trainer Filters to Schedule

Three independent, AND-composed filters on both the staff calendar and the public embed timetable. Filtering is instant (client-side over already-loaded data — no loader refetch on filter change).

## What Was Built

### Task 1 — Staff calendar (`gymos.schedule.tsx`)

**Loader Query A widened:**
- Added `.leftJoin(schema.trainers, eq(schema.classOccurrences.trainerId, schema.trainers.id))` after the existing classDefinitions join
- Added `location`, `trainerId`, and `trainerName` to the select object
- `// guard:allow-unscoped` comment already present; noted that new trainers join is covered by the same single-tenant exemption

**Filter state (client-side, no URL persistence):**
- `useState<string>` for each: `locationFilter`, `classTypeFilter`, `trainerFilter` — all default `"all"`
- `locationOptions` derived from distinct non-null `o.location` values in loaded occurrences (sorted)
- `classTypeFilter` matches on `o.className`; options from `data.classTypes` (stable even on months with no occurrences)
- `trainerFilter` matches on `o.trainerId` (stable ID, not name string); options from `data.trainers`
- `filteredOccurrences` = AND-compose all three filters before `groupByDay`
- `byDay` is now computed from `filteredOccurrences` — day-cell counts and day-detail list both narrow

**Filter UI (progressive disclosure):**
- Single `Popover` trigger `Button variant="outline" size="sm"` with `IconFilter` (Tabler) + count badge when any filter active
- Inside Popover: three labelled shadcn `Select` controls (Location / Class type / Trainer) + "Clear all filters" ghost button
- Placed in existing header control cluster alongside ManageTrainersDialog / NewClassDialog / Share Popover
- No always-visible selects cluttering the header

**Empty-after-filter state:**
- Day detail pane branches on `hasActiveFilters`: when true and `selectedOccurrences.length === 0`, shows "No classes match your filters" + "Clear filters" Button
- When `hasActiveFilters` is false, retains original "No classes scheduled / Pick another day" copy

**Also:** location and trainerName are now surfaced in the occurrence card detail row (previously unshown).

### Task 2 — Public embed (`schedule-widget-ssr.ts`)

**Query widened:**
- `ClassRow` type extended with `location: string | null` and `trainerName: string | null`
- `.leftJoin(schema.trainers, eq(schema.classOccurrences.trainerId, schema.trainers.id))` added after existing `.innerJoin(classDefinitions, ...)`
- `location` and `trainerName` added to select
- Pre-existing `// guard:allow-unscoped` comment covers trainers join

**Data attributes on class cards:**
- `data-location`, `data-classname`, `data-trainer` on each `.class-card` div (HTML-escaped; null → empty string)
- Location + trainer name rendered as `.class-loc` / `.class-trainer` chips in card details (same visual style as `.class-cat`/`.class-dur`/`.class-room`)

**Filter bar (server-side rendered, plain HTML):**
- `renderFilterBar(classes)` derives distinct option lists from loaded classes (server-side)
- Three native `<select>` elements with "All …" first option
- `#filter-empty` div (hidden by default) for the no-match state
- Placed above `renderSchedule()` output

**Inline JS (`applyFilters`):**
- Reads the three `<select>` values; shows/hides `.class-card` elements via AND logic
- Empty filter value = match-all; `data-location === ""` occurrences always visible under "All locations"
- Hides `.day-group` elements when all their cards are hidden (no empty day headers)
- Toggles `#filter-empty` visibility when `visibleCount === 0`
- Calls `sendResize()` after filtering so parent iframe re-fits height
- Each `<select>` wired via `onchange="applyFilters()"`

**CSS additions:**
- `.filter-bar` — flex-wrap row, 8px gap, 20px margin-bottom
- `.filter-select` — flex 1 1 140px, native appearance, accent border on focus
- `.filter-empty` — centered, muted text (distinct from `.empty-state` which covers zero scheduled classes)
- `.class-loc`, `.class-trainer` — same chip style as `.class-cat`
- Mobile: `.filter-bar` stacks vertically, `.filter-select` full width
