<objective>
Merge the staff inbox's "Messages" and "Leads" tabs into ONE unified conversation list (no tabs, no leads filter), and render a subtle per-conversation source badge that calls out where each lead came from — the source TYPE for all leads, plus the specific FORM NAME for form leads ("Form: {title}").

Purpose: The split Messages/Leads view forces staff to context-switch between two lists for what is really one stream of conversations. Merging them into a single list with an at-a-glance source badge gives coaches the full picture without tab-hopping, and tells them instantly whether a new conversation came from a form, an import, a Meta ad, or organic WhatsApp.

Output: A single modified route file. No schema migration — the form name is derived from existing `form_submissions` → `forms.title`; the source type from `whatsapp_opt_in.source`. (User decisions locked: D-01 type + form name; D-02 fully merged, no leads filter.)
</objective>
