# Phase quick-260625-x34 Plan 01: Merge Messages/Leads Inbox into One List ‚Äî Summary

**One-liner:** Merged WhatsApp inbox and leads list into a single unified conversation stream with per-lead source badges (form title, import, Meta ad, WhatsApp, manual) resolved from existing opt-in + form_submissions tables ‚Äî no schema migration.

## What Was Built

The staff `/gymos/messages` inbox previously split conversations into two separate views: a "Messages" list (non-lead conversations) and a "Leads" tab (status='lead'). Coaches had to flip between tabs to see the full conversation stream.

This plan collapsed both lists into one, and added subtle per-lead source badges so coaches instantly see where each lead came from (e.g. "Form: Hyrox Enquiry" or "Imported").

### Loader changes (Task 1)

- Removed the `isLeadsView` partition and the `ne(status,'lead')` / `eq(status,'lead')` WHERE clause. The loader now returns ALL conversations ordered by `updatedAt DESC`.
- Removed `ne` from the drizzle-orm import (became unused); `eq` retained.
- Added a two-query `sourceMap` fan-out:
  - Query (a): joins `conversations ‚Üí whatsapp_opt_in` via `member_id` to get the `source` enum value per lead conversation. Maps enum to human labels (`form_submission ‚Üí "Form"`, `import ‚Üí "Imported"`, etc.).
  - Query (b): `DISTINCT ON (fs.conversation_id)` join of `form_submissions ‚Üí forms` to get the form title per lead. Overrides "Form" with "Form: {title}" (e.g. "Form: Hyrox Enquiry"). Guards `leadConvIds.length > 0` to avoid an empty `IN ()`.
  - Both queries carry `// guard:allow-unscoped ‚Äî coach inbox shows all conversations in the studio`.
- Each conversation row is enriched with `leadSource: { type, label } | null`. Non-leads always get `null`.
- `isLeadsView` removed from the return value.

### Component changes (Task 2)

- Removed `const { isLeadsView } = data;` from the component.
- Added `sourceIcon()` module-level helper that maps source type ‚Üí Tabler icon component.
- Header h1 changed from `{isLeadsView ? "Leads" : "Messages"}` to `"Inbox"`.
- `<ImportLeadsDialog>` made unconditional (was gated on `isLeadsView`).
- Deleted the Messages/Leads filter chip block (`<div className="flex items-center gap-1 mt-2">` with two Link chips) entirely.
- Removed the `{!isLeadsView && (...)}` wrapper from the "Conversations" section label ‚Äî now always rendered.
- Fixed the per-row Link `to` URL: removed `${data.isLeadsView ? "&filter=leads" : ""}` suffix.
- Fixed the thread back-nav Link: `to="/gymos/messages"` (dead `?filter=leads` removed).

### Source badge (Task 3)

Placed immediately after the D-20 window-state span in each conversation row. Uses shadcn `Badge` with `variant="secondary"` + `text-muted-foreground` for quiet visual weight. The IIFE pattern `{c.leadSource && (() => { ... })()}` resolves the icon component from the type and renders it inline. Label truncated at `max-w-[140px]` to protect the 320px rail.

Icons used:
- `IconForms` ‚Äî form_submission leads
- `IconUpload` ‚Äî imported leads
- `IconBrandWhatsapp` ‚Äî inbound WhatsApp leads
- `IconBrandMeta` ‚Äî Meta lead ads
- `IconUser` ‚Äî manually added leads / fallback

Non-lead conversations render no badge (`c.leadSource` is `null`).
