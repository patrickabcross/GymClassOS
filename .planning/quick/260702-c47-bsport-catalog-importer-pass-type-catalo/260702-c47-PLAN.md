<objective>
Add a pass-type catalog to RunStudio (apps/staff-web) so a studio owner defines their own pass products and each pass type's category-based class compatibility, entered via a staff catalog UI — replacing the blocked bSport CSV import. Booking then enforces that the pass paying for a class allows that class's category, while legacy passes (null pass_type_id) keep working as allow-all.

Purpose: Prove RunStudio can model bSport-style "this pass books these classes" rules without bSport, repeatable per client (no HUSTLE hardcoding).
Output: `pass_types` catalog table + `passes.pass_type_id` link, three catalog actions, category enforcement wired into the correctness-critical booking transaction, and a "Catalog" staff surface with agent nav exposure.
