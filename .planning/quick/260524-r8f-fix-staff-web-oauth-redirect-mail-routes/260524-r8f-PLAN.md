<objective>
Fix staff-web post-login UX on gym-class-os.vercel.app so that:

1. Logging in with Google lands the user on /gymos (the GymOS WhatsApp inbox) — not on /inbox showing the Mail-template UI with real Gmail data.
2. Google OAuth consent only asks for profile + email scopes, not Gmail/Calendar/Contacts.
3. The framework's Mail-template account hook (Gmail watch) is removed so it cannot run on sign-in.

The Vercel deploy fix (pg externalization in vite.config.ts + post-vercel-build.mjs hook) is load-bearing for production and MUST be left untouched.

Purpose: Demo-blocking — production URL currently shows another user's Gmail inside the Mail UI for anyone who signs in. Fix is small and surgical.

Output:
- Two route files converted from InboxPage renderers into loader+clientLoader redirects to /gymos.
- One library file edit narrowing SCOPES and removing the Gmail-watch account hook from the OAuth exchange path.
- One plugin file edit removing the Gmail-watch renewal cron call (defence in depth — the env-gate already short-circuits when GMAIL_WATCH_TOPIC is unset on production, but removing the call site is cleaner).
</objective>
