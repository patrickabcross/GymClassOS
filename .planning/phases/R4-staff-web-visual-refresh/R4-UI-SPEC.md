## Spacing Scale

Multiples of 4px only. Tailwind v4 spacing utilities map to these.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, inline badge padding, dot indicators |
| sm | 8px | p-2 / gap-2 | Compact element spacing, card inner gaps |
| md | 16px | p-4 / gap-4 | Default element spacing, card padding |
| lg | 24px | p-6 / gap-6 | Section padding, panel headers |
| xl | 32px | p-8 / gap-8 | Layout column gaps |
| 2xl | 48px | p-12 | Major section breaks |
| 3xl | 64px | p-16 | Page-level outer spacing (rarely used) |

Exceptions:
- Touch targets for icon-only controls: minimum 44px height/width (h-11 w-11). Applied to any icon-button in the top nav or card actions so coaches using phones on the gym floor can tap reliably (D-06 mobile coaching context).
- Class card inner padding: 12px (p-3) to keep the card compact on a day-detail list.
- Nav tab padding: 10px horizontal / 4px vertical (px-2.5 py-1) — matches existing GymosTopNav.

## Typography

Inter variable font, self-hosted. Four roles, two weights max.

| Role | Size | Weight | Line Height | Tailwind classes | Usage |
|------|------|--------|-------------|-----------------|-------|
| Body | 13px | 400 (regular) | 1.5 | text-[13px] font-normal | Message previews, timeline descriptions, form labels |
| Label | 12px | 600 (semibold) | 1.4 | text-[12px] font-semibold | Card metadata, nav tabs (active), badge labels, table column headers |
| Heading | 14–16px | 600 (semibold) | 1.3 | text-sm font-semibold / text-base font-semibold | Class name on card, member name, panel section headers |
| Display | 20px | 700 (bold) | 1.2 | text-xl font-bold | Pass balance number only ("8" in "8 credits") — not used elsewhere |

Notes:
- No third weight; 400 and 600 cover all cases. 700 appears only on the pass-balance numeral (the one operational metric coaches read at a glance).
- Uppercase tracking (`text-[11px] uppercase tracking-wide`) is reserved for two cases: weekday abbreviations in the schedule header (Mon/Tue...) and sub-labels in widget cards (e.g. "PASS BALANCE"). Not for body copy.
- All numeric data (booked counts, credits, times) uses `tabular-nums` to prevent layout jitter as values change.
- Muted sub-text (secondary metadata, timestamps): `text-muted-foreground text-[11px]`.

## Color

All values expressed as token roles. Executor must use CSS variable-driven utility classes. Raw hex is CI-blocked.

| Role | Token | Default skin value (reference only) | Usage |
|------|-------|-------------------------------------|-------|
| Dominant (60%) | `--background` | hsl of near-white | Page background, main content area |
| Secondary (30%) | `--card` / `--muted` | hsl of white / light slate | Cards, nav strip, member context panel, schedule grid cells |
| Accent (10%) | `--studio-accent` / `--primary` | orange-500 (#F97316 reference) | See reserved-for list below |
| Destructive | `--destructive` | red-600 range | Destructive actions only: cancel booking dialog confirm, pass revoke (if added) |

**Accent (`--studio-accent` / `--primary`) is reserved exclusively for:**
1. Active nav tab background and text (the currently active route pill)
2. Capacity-full status: the capacity badge when 0 spots remain (red semantic — use `--destructive` token, not accent; see capacity states below)
3. Primary action Button (`variant="default"`) — "Book" CTA, "Send Enquiry" submit, "New Message" primary button
4. Focus rings on interactive elements (via Tailwind `ring-primary/20`)
5. "Today" date highlight in the schedule month grid
6. Pass balance numeral display (`text-primary` or `text-[--studio-accent]`)

**Semantic capacity color states** (SWEB-02 — the single most-watched UAT detail):

| State | Spots remaining | Background token | Text token | Tailwind pattern |
|-------|----------------|-----------------|-----------|-----------------|
| Normal | > 3 spots | transparent | `text-muted-foreground` | `text-muted-foreground` |
| Amber (warning) | 1, 2, or 3 spots | `bg-amber-100` | `text-amber-700` | `bg-amber-100 text-amber-700` — Note: amber is NOT a shadcn token; this is an intentional semantic exception. Allowlist in guard: `// guard:allow-color — capacity-warning amber semantic, not a brand color` |
| Red (full) | 0 spots | `bg-destructive/10` | `text-destructive` | `bg-destructive/10 text-destructive` |

The capacity color is applied to the Badge component only — not the card border or card background. Card background stays `--card` regardless of capacity state (clean screen, no clutter).

**Light theme only.** Dark toggle removed (D-08). The `.dark` CSS class block is left dormant in `global.css` (lowest-risk for R2 skin cascade; do not remove). The `ThemeToggle.tsx` component is removed from the nav. `next-themes` ThemeProvider stays in root.tsx but defaults to `light` with no toggle exposed. Post-milestone: DSGN-F1.

**Portal inheritance:** All Radix portals (Dialog, Tooltip, Popover, Select, DropdownMenu) inherit studio tokens because `data-studio` is on `<html>`, not an inner div. Do not move `data-studio` — this is a hard invariant (R-14).

## Surface Contracts

### 1. Schedule (SWEB-01, SWEB-02)

**Layout:** Two-pane split — month grid on the left, day-detail class list on the right. Existing layout in `gymos.schedule.tsx` is correct; this spec extends the class card anatomy in the right pane.

**Class card anatomy (right pane — day detail list):**

```
┌─────────────────────────────────────────────┐  ← shadcn Card, border-border/50, bg-background/60, p-3
│  09:00 – 10:00          [capacity badge]    │  ← time: text-[12px] font-semibold tabular-nums; badge: right-aligned
│  Boxing Fundamentals                        │  ← class name: text-[13px] font-semibold, mt-1
│  ──────────────────────────────────────────  │
│  [Boxing] [Room A] · 60min    [Book] or     │  ← category Badge variant="secondary" h-4 text-[9px] uppercase
│                                [Full]       │  ← Book: Button size="sm" variant="default" h-6 px-2 text-[11px]
└─────────────────────────────────────────────┘     Full: Button size="sm" variant="outline" disabled
```

**Capacity badge (upper-right of card):**
- Render as inline `<span>` with `tabular-nums` — NOT a shadcn Badge (Badge is for status pills; the booked count is inline data).
- Text format: `{booked} / {capacity}` — always shown.
- Color: apply capacity states from the Color section above. Transition from muted-foreground (normal) → amber-700 (≤3) → destructive (0 spots).
- When full: append " · Full" to the text; disable the Book button.

**Instructor field:** The schema has `class_definitions` but no explicit instructor column confirmed from the existing loader. Do NOT add a stub "Instructor: TBD" string. Show the instructor only when the data field is non-null. If instructor data is absent from the current schema, omit the field entirely from the card — clean screen over showing a placeholder. (Planner: verify instructor field availability at plan time; if absent, skip it from SWEB-01 for v1.1.)

**Empty state (no classes on selected day):**
- Container: `flex flex-col items-center justify-center h-full gap-2 px-4 py-8 text-center`
- Icon: `<IconCalendarEvent size={28} className="text-muted-foreground" aria-hidden />` (Tabler — already in existing code)
- Heading: "No classes scheduled" — text-[13px] text-muted-foreground
- Body: "Pick another day from the calendar" — text-[11px] text-muted-foreground
- No CTA in empty state (adding a class is an admin action; keep the schedule surface clean)

**Month grid day cells:** Existing implementation is acceptable. One visual refinement: today's date cell uses `bg-studio-accent/10 border-studio-accent/30` (via `[--studio-accent]` token) to replace the current `border-foreground/40`. Gives it the studio accent glow rather than a generic black border.

**Cancelled class cards:** Existing `opacity-50` treatment is correct. Add a "Cancelled" Badge in `variant="outline"` h-4 text-[9px] uppercase at the end of the metadata row (already in code; keep it).

**Visual acceptance criteria (UAT — deploy-only verification):**
- A class with 0 spots shows red destructive text on the booked/capacity display
- A class with 1, 2, or 3 spots shows amber text with amber-100 background
- A class with 4+ spots shows muted-foreground text
- The "Book" button is absent/disabled on full and cancelled classes

### 5. Messages Responsiveness (SWEB-06)

**Desktop layout (≥ 768px):** Three-pane side-by-side:
```
[Conversation list | 320px fixed] [Thread | flex-1] [Member context panel | 260px fixed]
```
- The conversation list and member context panel have fixed widths; the thread takes remaining space
- All three panes scroll independently (`overflow-y-auto`)

**Mobile layout (< 768px):** Single-column. Only the conversation list is visible by default.

Tapping a conversation:
1. The thread takes full-width (conversation list slides out / is hidden)
2. A persistent "Member" icon-button appears in the thread header (top-right area) — `<IconUser size={18} />` in a `Button variant="outline" size="icon" h-9 w-9`
3. Tapping that button opens the Member Context Sheet

**Bottom Sheet trigger (mobile member context):**

Use `shadcn Sheet` with `side="bottom"`.
```tsx
<Sheet>
  <SheetTrigger asChild>
    <Button variant="outline" size="icon" className="h-9 w-9">
      <IconUser size={18} aria-hidden />
      <span className="sr-only">Member context</span>
    </Button>
  </SheetTrigger>
  <SheetContent side="bottom" className="h-[60vh] overflow-y-auto rounded-t-xl">
    {/* Same widget card stack as the desktop context panel */}
    {/* Avatar + name, Pass Balance card, Next Class card, Last Visit card */}
    {/* View Member Profile button */}
  </SheetContent>
</Sheet>
```

- Sheet height: `h-[60vh]` — shows enough to scan all three widget cards without obscuring the header
- Sheet rounded top: `rounded-t-xl` (0.75rem, within the `--radius: 0.5rem` skin default — adjust to `rounded-t-[calc(var(--radius)+0.25rem)]` for skin correctness)
- Sheet has a drag handle (shadcn default) — no custom dismiss affordance needed
- Content inside the Sheet is identical to the desktop context panel widget stack

**Breakpoint:** Tailwind `md` breakpoint (768px). Below `md`, hide the context panel (`hidden md:block`). Above `md`, hide the Sheet trigger button (`md:hidden`).

**Back navigation on mobile:** When a thread is open in single-column mode, show a "← Messages" back button above the thread header. This navigates back to the conversation list without a full page reload (URL-state-driven, same `?conversation=` param pattern).

**SWEB-08: Light theme default, dark toggle removed.**

- Remove `<ThemeToggle />` component from the nav (if currently rendered there)

- `next-themes` package stays installed — only the toggle UI is removed
- Visual acceptance criteria: no dark-mode switch appears anywhere on the staff web; hard-reloading any `/gymos/*` route renders in light theme regardless of system preference

### 7. Embed Widgets (WDGT-01, WDGT-02, WDGT-03)

Both embed surfaces live in `apps/staff-web/features/forms/lib/`. They are SSR-rendered standalone HTML pages served inside `<iframe>` tags. Iframe isolation is already correct (R-05 — no Shadow DOM work required per REQUIREMENTS.md).

**Token theming in embeds:**

The schedule widget SSR already injects `--gym-accent` and `--gym-radius` from sanitized URL params. R4 extends this to the full `--studio-*` token set:
- URL params: `?accent=<hex>` → `--studio-accent`; `?radius=<0-32>` → `--radius`; `?soft=<hex>` → `--studio-accent-soft`
- All hex params are sanitized by the existing `sanitizeHexColor()` function before injection
- Fallback when params absent: use `--studio-accent: #F97316` (orange-500 default) and `--radius: 0.5rem`
- The `data-studio` attribute is NOT applied inside the iframe (no staff-web skin injector inside the SSR HTML) — token values are injected directly into `<style>` as `:root { --studio-accent: ...; }` inline CSS inside the iframe's `<head>`
- Font: Inter already self-hosted; the `/fonts/inter-variable.woff2` reference is same-origin so the iframe's `<link>` preload works without cross-origin issues (per R2-CONTEXT.md D-13)

#### /embed/schedule (WDGT-01)

**Layout:** Vertical card list, no admin chrome. No top nav, no sign-out, no admin buttons.

```

└─────────────────────────────────────────────────────────────────┘
```

Quick reference for executor. All patterns use CSS variable-driven Tailwind utilities only.

## Interaction Constraints (Hard)

1. **No custom dropdowns/modals.** Any dropdown must be `<DropdownMenu>` from shadcn; any modal must be `<Dialog>` or `<AlertDialog>`. No `position: absolute` + click-outside `useEffect`.
2. **No emojis as icons.** Use Tabler icons for all iconography. A gym class name that contains an emoji (user content) is acceptable; UI chrome never uses emoji.
3. **No browser dialogs.** `window.confirm`, `window.alert`, `window.prompt` are forbidden. Use `<AlertDialog>` for destructive confirmations.
4. **Progressive disclosure.** Cards show summary; details reveal on expand/click. Bookings timeline: show first 5, Collapsible reveals rest. Context panel: three widget cards, no table dump.
5. **Optimistic UI on all mutations.** The "Book" action in the schedule dialog must optimistically update the booked count before the server responds. No blocking spinner on primary CTAs. Roll back + toast on error.
6. **Portal inheritance.** All Radix portals (Dialog, Tooltip, Popover, Select content) render at `document.body` but inherit `data-studio` tokens because the attribute is on `<html>`. Do not add workarounds.
7. **Clean important screens.** No toolbar clutter. The schedule day-detail pane shows class cards and nothing else. The member context panel shows three widget cards and nothing else. Do not add filters, badges, or helper text that isn't directly operational.
